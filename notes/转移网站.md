## 步骤 6：配置 Nginx

```bash
# 创建 Nginx 配置文件
sudo vim /etc/nginx/sites-available/zhangyiyang.xyz
```

**配置内容要点：**
- `listen 443 ssl;`
- `ssl_certificate` 指向 `/root/zhangyiyangxyz/certificate/zhangyiyang.xyz_bundle.pem`
- `ssl_certificate_key` 指向 `/root/zhangyiyangxyz/certificate/zhangyiyang.xyz.key`
- `location ^~ /files` 代理到 `http://127.0.0.1:5000`
- `location /` 的 root 为 `/root/zhangyiyangxyz/docs`
- HTTP (80端口) 重定向到 HTTPS

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/zhangyiyang.xyz /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default  # 删除默认配置

# 测试并重启
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl enable nginx
```

## 步骤 7：配置 systemd 服务（详细版）

systemd 服务配置可以让 Flask 应用在后台持续运行，并在服务器重启后自动启动。

---

### 7.1 创建 systemd 服务文件

```bash
# 在服务器上执行
sudo vim /etc/systemd/system/flask-netdisk.service
```

```ini
[Unit]
Description=Flask NetDisk Application for zhangyiyang.xyz
After=network.target

[Service]
Type=simple
User=root
Group=root
WorkingDirectory=/root/zhangyiyangxyz/docs/Tools/NetDisk
Environment="PATH=/usr/local/bin:/usr/bin:/bin"
ExecStart=/usr/bin/python3 /root/zhangyiyangxyz/docs/Tools/NetDisk/NetDisk_for_zhangyiyang_xyz.py
Restart=always
RestartSec=3

# 日志配置
StandardOutput=journal
StandardError=journal
SyslogIdentifier=flask-netdisk

[Install]
WantedBy=multi-user.target
```

**配置说明：**

### `[Unit]` 部分
- `Description=` - 服务的描述信息
- `After=network.target` - 在网络服务启动后再启动此服务

### `[Service]` 部分
- `Type=simple` - 简单的服务类型
- `User=root` - 以 root 用户身份运行
- `Group=root` - 使用 root 用户组
- `WorkingDirectory=` - Flask 应用的工作目录
- `Environment=` - 设置环境变量（Python 路径）
- `ExecStart=` - 启动命令（完整的 Python 解释器路径 + 脚本路径）
- `Restart=always` - 崩溃后自动重启
- `RestartSec=3` - 重启前等待 3 秒
- `StandardOutput=journal` - 将标准输出发送到 systemd 日志
- `StandardError=journal` - 将错误输出发送到 systemd 日志
- `SyslogIdentifier=` - 日志标识符

### `[Install]` 部分
- `WantedBy=multi-user.target` - 在多用户模式下启动

---

### 7.2 确认 Python 路径（重要）

在创建服务前，确认 Python 3 的实际路径：

```bash
# 查找 Python 3 的路径
which python3

# 应该输出类似：/usr/bin/python3
```

如果输出的路径不是 python3，需要修改服务文件中的 `ExecStart` 路径。

---

### 7.3 重新加载 systemd 配置

每次修改 systemd 服务文件后，都需要重新加载配置：

```bash
sudo systemctl daemon-reload
```
